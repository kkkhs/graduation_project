## 1. 遥感图像基础

### 1.1 什么是遥感图像

**遥感**：通过非接触的方式（如卫星、飞机、无人机）获取地球表面信息的技术。

**遥感图像**：通过遥感设备拍摄的地球表面图像。

### 1.2 遥感图像的特点

| 特点         | 说明                                 | 对检测的影响                     |
| ------------ | ------------------------------------ | -------------------------------- |
| **分辨率**   | 分为高分辨率、中分辨率、低分辨率     | 分辨率越低，目标越小，检测越困难 |
| **多光谱**   | 包含多个波段（红、绿、蓝、近红外等） | 可以利用多光谱信息提升检测效果   |
| **大视场**   | 单张图像覆盖范围大                   | 目标密度低，背景复杂             |
| **几何畸变** | 存在透视畸变、地形起伏影响           | 需要几何校正                     |
| **大气影响** | 云、雾、光照变化                     | 影响图像质量和目标可见性         |

### 1.3 分辨率分类

| 类型         | 空间分辨率 | 典型卫星                        | 应用场景                       |
| ------------ | ---------- | ------------------------------- | ------------------------------ |
| **高分辨率** | < 1 米     | WorldView、QuickBird            | 城市规划、精细目标检测         |
| **中分辨率** | 1-10 米    | Landsat、Sentinel-2、Gaofen-1/6 | 土地利用、农作物监测、船舶检测 |
| **低分辨率** | > 10 米    | MODIS、AVHRR                    | 气象监测、海洋监测             |

**LEVIR-Ship 数据集**：中分辨率遥感图像（16m/pixel）

### 1.4 常见遥感卫星

| 卫星            | 分辨率   | 重访周期 | 应用                   |
| --------------- | -------- | -------- | ---------------------- |
| **Landsat 8/9** | 30 米    | 16 天    | 土地利用、环境监测     |
| **Sentinel-2**  | 10-60 米 | 5 天     | 农业、森林监测         |
| **WorldView-3** | 0.31 米  | 1 天     | 精细目标检测           |
| **Gaofen-1**    | 16 米    | 4 天     | 中国遥感应用、船舶检测 |
| **Gaofen-6**    | 16 米    | 4 天     | 中国遥感应用、船舶检测 |

## 2. 目标检测基础

### 2.1 什么是目标检测

**目标检测**：在图像中定位并识别感兴趣的目标。

**输出**：

- **边界框**：目标的矩形框
- **类别**：目标的类别（如"船舶"）
- **置信度**：检测的可信度

### 2.2 目标检测的发展历程

```
传统方法（2012年前）
    ↓
深度学习兴起（2012-2015）
    ↓
两阶段检测器（2015-2017）
    ↓
单阶段检测器（2016-2018）
    ↓
Anchor-Free 检测器（2019-2020）
    ↓
Transformer 检测器（2020-至今）
```

### 2.3 目标检测分类

| 分类            | 代表方法     | 特点                         |
| --------------- | ------------ | ---------------------------- |
| **两阶段**      | Faster R-CNN | 先生成候选框，再分类和回归   |
| **单阶段**      | YOLO、SSD    | 直接预测边界框和类别         |
| **Anchor-Free** | FCOS         | 不使用预设锚框，逐像素预测   |
| **Transformer** | DETR、DINO   | 使用注意力机制进行端到端检测 |

### 2.4 边界框表示

**水平边界框**：

```
[x_min, y_min, x_max, y_max]
或
[x_center, y_center, width, height]
```

**旋转边界框**：

```
[x_center, y_center, width, height, angle]
```

## 3. 小目标检测的挑战

### 3.1 什么是小目标

**定义**：在图像中占据像素数很少的目标。

**COCO 数据集分类**：

- **小目标**：面积 < 32² 像素
- **中目标**：32² < 面积 < 96² 像素
- **大目标**：面积 > 96² 像素

**LEVIR-Ship 中的微小目标**：在 GF-1 图像（16m/pixel）中，船舶可能仅占 20 像素

### 3.2 小目标检测的难点

| 挑战           | 说明                     | 解决思路                 |
| -------------- | ------------------------ | ------------------------ |
| **特征少**     | 目标像素少，特征信息不足 | 多尺度特征融合、超分辨率 |
| **背景干扰**   | 目标容易被背景淹没       | 注意力机制、特征增强     |
| **漏检率高**   | 小目标容易被忽略         | 损失函数设计、数据增强   |
| **误检率高**   | 噪声容易被误认为目标     | 后处理、阈值调整         |
| **尺度变化大** | 同类目标尺寸差异大       | 多尺度训练、FPN          |

### 3.3 小目标检测常用技术

| 技术           | 说明                 | 代表方法           |
| -------------- | -------------------- | ------------------ |
| **特征金字塔** | 多尺度特征融合       | FPN、PAFPN         |
| **注意力机制** | 突出重要特征         | CBAM、SENet、CRMA  |
| **超分辨率**   | 提升目标分辨率       | SRGAN、ESRGAN      |
| **数据增强**   | 增加小目标样本       | Mosaic、Copy-Paste |
| **损失函数**   | 解决样本不平衡       | Focal Loss         |
| **退化重建**   | 通过重建引导特征学习 | DRENet 的 DRE 模块 |

## 4. 船舶检测的特点

### 4.1 船舶检测的应用场景

| 场景             | 说明                   |
| ---------------- | ---------------------- |
| **海上交通监控** | 监测船舶航行、防止碰撞 |
| **非法捕捞检测** | 发现非法捕捞行为       |
| **走私监控**     | 监控海上走私活动       |
| **搜救行动**     | 协助海上搜救           |
| **海洋污染监测** | 监测油轮泄漏           |

### 4.2 船舶检测的挑战

| 挑战           | 说明                     |
| -------------- | ------------------------ |
| **目标小**     | 船舶在遥感图像中通常很小 |
| **背景复杂**   | 海面波浪、云层、岛屿干扰 |
| **密度低**     | 单张图像中船舶数量少     |
| **尺度变化大** | 不同类型船舶尺寸差异大   |
| **运动模糊**   | 船舶运动导致图像模糊     |
| **海面反光**   | 阳光反射影响检测         |
| **碎云干扰**   | 碎云容易被误认为船舶     |

## 5. 深度学习基础回顾

### 5.1 卷积神经网络（CNN）

**卷积层**：提取局部特征

```
输入图像 → 卷积 → 激活 → 池化 → ...
```

**常用组件**：

- **卷积层**：提取特征
- **池化层**：降采样，减少参数
- **激活函数**：ReLU、GELU
- **归一化**：BatchNorm、LayerNorm

### 5.2 残差网络（ResNet）

**核心思想**：使用跳跃连接解决深度网络退化问题。

```
x → [卷积层] → F(x) → F(x) + x → 输出
```

**优势**：

- 可以训练更深的网络
- 梯度传播更稳定

### 5.3 特征金字塔网络（FPN）

**核心思想**：融合不同尺度的特征。

```
高分辨率特征（浅层）→ 侧向连接
中分辨率特征（中层）→ 侧向连接
低分辨率特征（深层）→ 上采样
                    ↓
              融合特征
```

**优势**：

- 同时检测不同大小的目标
- 提升小目标检测能力

### 5.4 注意力机制

**通道注意力**：关注重要的特征通道

**空间注意力**：关注重要的空间位置

**CBAM**：结合通道注意力和空间注意力

```
输入 → 通道注意力 → 空间注意力 → 输出
```

**CRMA（跨阶段多头注意力）**：

- 结合多头自注意力（MHSA）
- 获得大感受野
- 节省计算量

### 5.5 通道注意力块（RCAB）

**Residual Channel Attention Block**：

- 残差连接 + 通道注意力
- 用于 DRENet 的退化重建增强器
- 提升特征表达能力

## 7. 评价指标

### 7.1 基础指标

**TP（True Positive）**：正确检测到的目标

**FP（False Positive）**：错误检测到的目标（误检）

**FN（False Negative）**：未检测到的目标（漏检）

**TN（True Negative）**：正确忽略的背景

### 7.2 Precision（精确率）

```
Precision = TP / (TP + FP)
```

**含义**：检测到的目标中有多少是正确的

### 7.3 Recall（召回率）

```
Recall = TP / (TP + FN)
```

**含义**：所有目标中有多少被检测到

### 7.4 F1-Score

```
F1 = 2 * (Precision * Recall) / (Precision + Recall)
```

**含义**：精确率和召回率的调和平均

### 7.5 IoU（交并比）

```
IoU = 预测框与真实框的交集 / 预测框与真实框的并集
```

**含义**：预测框与真实框的重叠程度

### 7.6 AP（Average Precision）

**AP**：不同召回率下的精确率的平均值

**计算方法**：

- 在不同 IoU 阈值下计算 Precision-Recall 曲线
- 计算 PR 曲线下的面积

### 7.7 mAP（mean Average Precision）

**mAP**：所有类别的 AP 的平均值

**COCO 指标**：

- **AP50**：IoU=0.5 时的 AP
- **AP75**：IoU=0.75 时的 AP
- **AP50:95**：IoU 从 0.5 到 0.95（步长 0.05）的平均 AP
- **AP**：AP50:95 的简称

### 7.8 按尺寸分桶的 AP

**AP_small**：小目标的 AP

**AP_medium**：中目标的 AP

**AP_large**：大目标的 AP

**重要性**：小目标检测任务重点关注 AP_small

### 7.9 效率指标

**FPS（Frames Per Second）**：每秒处理的图像数

**Params（参数量）**：模型参数数量

**FLOPs（Floating Point Operations）**：浮点运算次数

**LEVIR-Ship 论文使用的指标**：AP、FPS、Params（推理）、FLOPs（推理）

## 8. 数据集和标注格式

### 8.1 数据集划分

| 划分       | 比例   | 用途                 |
| ---------- | ------ | -------------------- |
| **训练集** | 70-80% | 训练模型             |
| **验证集** | 10-15% | 调整超参数、选择模型 |
| **测试集** | 10-15% | 评估最终性能         |

**LEVIR-Ship 划分**：训练集 59.8%、验证集 20.3%、测试集 20.3%

### 8.2 COCO 标注格式

**JSON 结构**：

```json
{
  "images": [
    {
      "id": 1,
      "file_name": "image001.jpg",
      "width": 800,
      "height": 600
    }
  ],
  "annotations": [
    {
      "id": 1,
      "image_id": 1,
      "category_id": 1,
      "bbox": [x, y, width, height],
      "area": width * height,
      "iscrowd": 0
    }
  ],
  "categories": [
    {
      "id": 1,
      "name": "ship"
    }
  ]
}
```

**bbox 格式**：[x_min, y_min, width, height]

### 8.3 VOC 标注格式

**XML 结构**：

```xml
<annotation>
  <folder>images</folder>
  <filename>image001.jpg</filename>
  <size>
    <width>800</width>
    <height>600</height>
  </size>
  <object>
    <name>ship</name>
    <bndbox>
      <xmin>100</xmin>
      <ymin>200</ymin>
      <xmax>300</xmax>
      <ymax>400</ymax>
    </bndbox>
  </object>
</annotation>
```

### 8.4 YOLO 标注格式

**TXT 格式**：

```
class_id x_center y_center width height
```

**归一化**：x_center、y_center、width、height 都除以图像宽度和高度

**示例**：

```
0 0.5 0.5 0.1 0.2
```

### 8.5 数据格式转换

**常用工具**：

- **labelme2coco**：Labelme 转 COCO
- **voc2coco**：VOC 转 COCO
- **yolo2coco**：YOLO 转 COCO

## 9. 实验流程

### 9.1 完整实验流程

```
1. 数据准备
   ↓
2. 数据预处理
   ↓
3. 数据增强
   ↓
4. 模型选择
   ↓
5. 训练配置
   ↓
6. 模型训练
   ↓
7. 模型评估
   ↓
8. 结果分析
   ↓
9. 可视化展示
```

### 9.2 数据预处理

**步骤**：

1. 图像读取
2. 尺寸调整（统一输入尺寸）
3. 归一化（像素值归一化到 [0, 1]）
4. 数据格式转换（转换为框架所需格式）

### 9.3 数据增强

**常用方法**：

| 方法           | 说明               | 适用场景                  |
| -------------- | ------------------ | ------------------------- |
| **随机翻转**   | 水平/垂直翻转      | 增加样本多样性            |
| **随机旋转**   | 旋转图像           | 适应不同角度              |
| **颜色抖动**   | 调整亮度、对比度   | 适应不同光照              |
| **Mosaic**     | 拼接 4 张图像      | YOLO 系列，提升小目标检测 |
| **Copy-Paste** | 复制目标到其他图像 | 增加小目标样本            |
| **随机裁剪**   | 随机裁剪图像       | 增加尺度多样性            |
| **选择性退化** | 模拟图像退化       | DRENet 使用，突出目标     |

### 9.4 训练配置

**关键超参数**：

| 参数              | 说明         | 典型值             |
| ----------------- | ------------ | ------------------ |
| **batch_size**    | 批次大小     | 16、32             |
| **learning_rate** | 学习率       | 0.001、0.0001      |
| **epochs**        | 训练轮数     | 100、200、500      |
| **input_size**    | 输入图像尺寸 | 512×512、1024×1024 |
| **weight_decay**  | 权重衰减     | 0.0001、0.0005     |

**LEVIR-Ship 论文配置**：batch_size=16、epochs=500、input_size=512×512

### 9.5 训练监控

**监控指标**：

- **Loss**：训练损失和验证损失
- **mAP**：验证集上的 mAP
- **Learning Rate**：学习率变化
- **GPU 利用率**：显存占用、GPU 使用率

**工具**：TensorBoard、WandB

### 9.6 模型评估

**评估步骤**：

1. 在测试集上推理
2. 计算预测结果
3. 与真实标注对比
4. 计算评价指标（mAP、AP50 等）
5. 生成可视化结果

### 9.7 结果可视化

**可视化内容**：

- **成功案例**：检测正确的图像
- **失败案例**：漏检或误检的图像
- **难例**：难以检测的图像
- **对比图**：不同方法的对比结果

**LEVIR-Ship 论文可视化**：

- 蓝色框：正确检测（TP）
- 红色框：漏检（FN）
- 黄色框：误检（FP）

### 9.8 消融实验

**目的**：验证方法各部分的有效性

**LEVIR-Ship 论文消融实验**：

1. **退化函数 F(·) 的选择**
   - 线性函数
   - 对数函数
   - 指数函数（最佳）

2. **增强器结构设计**
   - 不同数量的 RCAB
   - 不同的上采样倍数
   - 结论：1个 RCAB + 2倍上采样效果最佳

3. **不同监督形式**
   - 直接下采样
   - 突然模糊 + 下采样
   - 连续模糊（Selective Degradation）+ 下采样（最佳）

### 11.4 DRENet 方法详解

#### 网络架构

DRENet 由三个主要部分组成：

**1. Backbone（骨干网络）**

- 基于 YOLOv5s（轻量级版本）
- 高效提取多尺度图像特征
- 输出特征图尺寸：16×16、32×32、64×64、64×64

**2. Degraded Reconstruction Enhancer（退化重建增强器）**

- 仅在训练阶段工作，推理时不产生计算开销
- 通过重建退化图像引导骨干网络关注微小船舶目标
- 使用单个 RCAB（Residual Channel Attention Block）
- 从 64×64 特征图重建 128×128 退化标签

**3. Detector（检测器）**

- 引入 CRMA（CRoss-stage Multi-head Attention）模块
- 利用自注意力机制提升特征判别能力
- 获得大感受野以准确定位船舶
- 节省计算量

#### Selective Degradation（选择性退化）

**核心思想**：生成一个"伪显著性图"，其中船舶保持完整，背景被模糊处理。

**算法流程**：

```
输入：图像 I，所有真实边界框集合 G
输出：退化图像 Î

对于图像中每个像素 (i, j)：
1. 计算该像素到所有船舶目标的最小距离 Dmin
2. 使用递增函数 F(Dmin) 调整均值滤波核大小
3. 应用相应大小的均值滤波
4. 对结果进行最近邻插值调整大小
```

**关键参数**：

- F(·)：递增函数，论文中使用指数函数效果最佳
- 核大小随距离增加而增大，离船舶越远模糊程度越高

## 12. 常见问题

### 12.1 训练相关

**Q：训练时显存不足怎么办？**
A：

- 减小 batch_size
- 减小输入图像尺寸
- 使用梯度累积
- 使用混合精度训练

**Q：训练不收敛怎么办？**
A：

- 检查学习率是否过大
- 检查数据标注是否正确
- 检查损失函数是否合适
- 尝试不同的优化器

**Q：过拟合怎么办？**
A：

- 增加数据增强
- 使用正则化（Dropout、Weight Decay）
- 增加训练数据
- 使用早停策略

### 12.2 检测相关

**Q：小目标漏检严重怎么办？**
A：

- 增加输入图像尺寸
- 使用特征金字塔（FPN）
- 增加小目标样本（Copy-Paste）
- 调整 NMS 阈值
- 尝试退化重建增强（如 DRENet 的 DRE 模块）

**Q：误检率高怎么办？**
A：

- 提高置信度阈值
- 使用更好的 NMS
- 增加负样本训练
- 使用难例挖掘

**Q：碎云导致误检怎么办？**
A：

- 使用选择性退化突出目标
- 增加注意力机制
- 增加碎云场景的训练数据
- 使用更好的特征提取网络

### 12.3 系统相关

**Q：如何部署模型？**
A：

- 导出 ONNX 格式
- 使用 TensorRT 加速
- 使用 Flask/FastAPI 封装 API

**Q：如何提升推理速度？**
A：

- 使用轻量级模型
- 使用模型量化（INT8）
- 使用 TensorRT 加速
- 批量推理
