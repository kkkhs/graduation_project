## 实验记录模板（每次训练/评测都建议生成一份）

> 目的：保证“可复现”，写论文第四章时直接引用该记录即可。

### 1. 实验信息
- **实验编号**：\<如 exp-2026-03-15-01\>
- **日期**：\<YYYY-MM-DD\>
- **模型/框架**：\<DRENet / mmdet-xxx / YOLO-xxx\>
- **代码版本**：\<git commit 或 tag\>
- **数据版本**：LEVIR-Ship \<版本/下载来源\>
- **划分方式**：\<官方划分/自定义划分，固定随机种子\>
- **随机种子**：\<seed\>

### 2. 训练设置
- **输入尺寸**：\<如 640 / 1024 / 多尺度\>
- **batch size**：\<填写\>
- **epoch/iters**：\<填写\>
- **优化器**：\<SGD/AdamW，lr 等\>
- **学习率策略**：\<cosine/step/warmup\>
- **数据增强**：\<mosaic / random crop / flip / color jitter 等\>
- **损失/后处理关键设置**：\<如 NMS iou 阈值、conf 阈值\>

### 3. 资源与耗时
- **硬件**：\<GPU 型号/显存，CPU，内存\>
- **训练耗时**：\<总耗时/每 epoch\>
- **显存峰值**：\<填写\>

### 4. 结果（定量）
- **mAP (AP50:95)**：\<数值\>
- **AP50**：\<数值\>
- **Precision/Recall/F1**：\<数值\>
- **备注**：\<是否 early stop，是否异常\>

### 5. 结果（定性）
- **可视化路径**：\<assets/figures/... 或输出目录\>
- **成功案例**：\<简述 2–3 条\>
- **失败案例/难例**：\<简述 2–3 条 + 可能原因\>

### 6. 复现命令

```bash
# train
<填写训练命令>

# eval
<填写评测命令>

# infer/visualize
<填写推理可视化命令>
```
