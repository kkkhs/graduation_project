# DRENet 本地复现补充（个人设备版）

> 本文档用于补充你本地机器的约束条件，不替代主指南。

## 1. 本地策略
1. 先“跑通优先”：小配置验证流程。
2. 再“结果优先”：正式配置产出主指标。
3. 最后“分析优先”：消融和难例整理。

## 2. 显存不足时的调整顺序
1. 降低 batch size。
2. 降低输入尺寸（如 512 -> 416）。
3. 使用梯度累积保持有效 batch。
4. 减少 dataloader workers。

## 3. 建议保留的固定项
- 固定随机种子
- 固定数据划分
- 固定评测脚本版本

## 4. 本地复现最小成功标准
- 训练可完成
- 评测可输出 AP50/Precision/Recall/F1
- 至少 6 张可视化样例图（成功2、漏检2、误检2）
