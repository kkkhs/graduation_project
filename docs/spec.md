## 本科毕业设计 Spec（任务清单）

### 项目信息
- **题目**：《遥感场景下的微小船舶检测系统设计与实现》
- **核心目标**：基于公开遥感船舶数据集，完成微小目标（船舶）检测的**对比实验**与**系统集成**，最终形成完整毕业论文与可运行系统原型。
- **硬性工作量要求（按老师口径整理）**
  - **国内外研究现状**：调研 **20–30 篇**相关论文（优先从 Survey / SOTA paper 的实验对比、Related Work 反向扩展）。
  - **模型对比**：至少 **3 种不同检测算法/实现**，在同一数据集上训练/测试，给出指标与可视化结果。
  - **系统实现**：集成至少 **3 种模型**，支持对遥感图像推理并输出检测结果（框/置信度/类别）与可视化。

### 参考资料（老师提供）
- **参考论文（tiny ship）**：Chen J, Chen K, Chen H, et al. *A degraded reconstruction enhancement-based method for tiny ship detection in remote sensing images with a new large-scale dataset*. IEEE TGRS, 2022.
- **数据集**：LEVIR-Ship（`https://github.com/WindVChen/LEVIR-Ship`）
- **参考代码（baseline）**：DRENet（`https://github.com/WindVChen/DRENet`）
- **目标检测工具库**：
  - mmdetection：`https://github.com/open-mmlab/mmdetection`
  - ultralytics（YOLO）：`https://docs.ultralytics.com/zh/`

## 任务总览（What to do）

### A. 文献调研与开题（优先级最高）
1) **确定调研范围与关键词（可扩展）**
   - **遥感微小目标检测**：remote sensing tiny object detection, tiny ship detection, small object detection
   - **检测范式**：one-stage (YOLO/RetinaNet)、two-stage (Faster R-CNN)、anchor-free (FCOS)、transformer-based (DETR/Deformable DETR/DINO)
   - **遥感检测常见点**：多尺度特征、背景复杂、目标稀疏、旋转框（如有）、超分/增强、去退化等

2) **收集 20–30 篇论文并分类整理**
   - **来源策略（按老师建议）**
     - 先找 **Survey/综述**，从其引用列表反向扩展
     - 找 2024/2025 近两年 SOTA paper，从其 **实验对比（comparison with SOTA）** 与 **related work** 反向扩展
   - **分类建议（写研究现状更顺）**
     - 通用目标检测方法演进（CNN/Transformer）
     - 小目标/微小目标专门方法（特征金字塔/注意力/超分/蒸馏等）
     - 遥感/船舶检测专门方法（数据集、成像退化、海面背景、尺度分布）
     - 工具链与工程化（mmdet/ultralytics 的可复现训练）
   - **交付物**
     - `docs/literature/`：论文清单（BibTeX/表格）+ 每篇 1 段摘要（问题-方法-数据-指标-结论）
     - “国内外研究现状”章节草稿（可直接进论文/开题）

3) **开题报告材料准备**
   - **问题定义**：遥感图像中的微小船舶目标检测（输入、输出、应用场景）
   - **技术路线**：数据集→基线复现→多模型对比→（可选）小改进/消融→系统集成→论文
   - **预期成果**：指标表、可视化、系统 Demo、论文

---

### B. 数据集与评价指标（实验可复现的地基）
4) **下载与理解 LEVIR-Ship 数据集**
   - 任务：明确数据组织结构、标注格式、类别定义、训练/验证/测试划分方式
   - 交付：`docs/dataset.md`（数据来源、许可、目录结构、统计信息：图像数/目标数/尺寸分布）

5) **统一标注格式与训练输入**
   - 任务：将数据集转换为你选择的训练框架格式（建议 **COCO** 作为中间格式）
   - 交付：
     - 数据转换脚本（可复现，固定随机种子）
     - 生成的索引/标注文件（或生成方式文档）

6) **评价指标定义**
   - 基础指标：Precision / Recall / F1
   - 检测指标：mAP（AP50、AP50:95）、PR 曲线
   - 可选（若目标极小）：按尺寸分桶（small/medium/large）统计 AP（若框架支持）
   - 交付：`docs/metrics.md`（指标定义、计算方式、使用工具）

---

### C. 复现参考论文方法（DRENet）并跑通全流程
7) **跑通 DRENet（训练 + 测试 + 推理可视化）**
   - 任务：
     - 复现仓库环境（依赖/权重/脚本）
     - 在 LEVIR-Ship 上训练与评估
     - 输出指标与可视化样例（成功/失败案例）
   - 交付：
     - `experiments/drenet/`：配置、命令、日志位置说明
     - 结果表（至少一组可复现的 checkpoint + 指标）

8) **复现实验记录规范化（后续写论文必需）**
   - 任务：固定随机种子、记录超参、记录训练时长/显存、记录数据版本
   - 交付：`docs/experiments/exp_log_template.md` + 每次实验一份记录

---

### D. 至少 3 个模型的对比实验（老师明确要求）
> 建议以“1个参考论文方法 + 2个通用检测器”为最稳妥组合。

9) **模型 1（论文/参考代码）：DRENet**
   - 输出：训练/测试指标 + 可视化

10) **模型 2（通用检测器，mmdetection 生态）**
   - 候选：Faster R-CNN / RetinaNet / FCOS / (Deformable) DETR 等（选 1 个）
   - 任务：用同一数据格式训练、统一增强与输入尺寸，给出对比结果
   - 输出：指标 + 可视化 + 训练配置文件

11) **模型 3（ultralytics YOLO 生态）**
   - 候选：YOLOv8/YOLOv11/YOLOv10 中选 1 个（建议先用最新稳定版本）
   - 任务：适配数据、训练、评估、导出推理权重
   - 输出：指标 + 可视化 + 导出模型（ONNX/pt 均可）

12) **对比实验表与分析**
   - 统一输入分辨率、训练轮数/早停策略、数据增强策略（或写清差异）
   - 表格输出：mAP、AP50、Recall、Params/FLOPs、FPS（可选）
   - 交付：`docs/results/baselines.md`（主对比表 + 结论）

13) **消融实验（至少 1–2 组，写论文第 4 章很关键）**
   - 方向示例（任选其一即可成章）
     - 输入尺寸/多尺度训练对微小目标的影响
     - 不同 NMS/置信度阈值对 Precision/Recall 的影响
     - 数据增强（Mosaic、Copy-Paste、随机裁剪）对小目标的影响
   - 交付：`docs/results/ablation.md`

14) **可视化结果集**
   - 任务：统一输出格式（原图 + 预测框 + 置信度），整理“成功/失败/难例”
   - 交付：`assets/figures/` + `docs/results/qualitative.md`

### E. 系统需求分析、设计与实现（“系统”是论文的一大章）
15) **需求分析（功能/非功能）**
   - 功能需求（建议最低集）
     - 选择模型（≥3）
     - 单张图片推理、批量推理
     - 输出：可视化图片、JSON/CSV 结果、指标评估（可选）
   - 非功能需求
     - 可复现安装、运行速度、GPU/CPU 兼容策略（写清“推荐配置/最低配置”）
   - 交付：`docs/system/requirements.md`

16) **系统总体设计（架构与模块）**
   - 模块建议
     - 数据层：数据读取与预处理
     - 模型层：统一推理接口（对接 DRENet/mmdet/YOLO）
     - 服务层：CLI 或 Web（Flask/FastAPI/Gradio 三选一）
     - 展示层：可视化与结果导出
   - 交付：`docs/system/design.md`（架构图 + 模块说明 + 关键接口）

17) **系统实现（最小可用版本 MVP）**
   - 任务：
     - 统一 `predict(image_path, model_name, conf, iou)` 风格接口
     - 推理结果保存与可视化
     - 一键运行脚本（如 `python -m app ...`）
   - 交付：可运行 Demo + `docs/system/user_guide.md`

18) **系统验收用例（答辩演示必须）**
   - 用例：3 张典型图（易/中/难），展示 3 个模型输出与差异
   - 交付：`docs/system/demo_checklist.md`
