# 毕设当前缺口盘点（2026-04-09）

> 盘点范围：系统实现、测试与交付、论文可提交性。
> 结论口径：以仓库当前文件与本地验证结果为准。

## 1. 当前已具备的完成度（可用于答辩与论文主线）

- 系统主链路已形成：React 前端 + FastAPI 后端 + SQLite 持久化 + 异步任务执行。
- 算法接入已具备：DRENet、YOLO26（主展示）+ FCOS（参考基线）可在统一入口下调度。
- 结果闭环已具备：任务提交、任务列表、任务详情、可视化结果与历史回看。
- 论文核心内容已形成：方法、主实验、消融、系统实现、总结章节均有正文与图表。
- 当前代码量（统计口径：`backend`、`frontend/src`、`src`、`tools`、`scripts`、`tests`）约 **7777 行**。

## 2. 硬缺口（建议优先处理）

### 2.1 论文本地编译字体依赖（已修复，仍建议做一次冷启动验收）

- 现状：已通过将模板字体集切换为更通用配置修复编译问题（不再依赖 `Kaiti SC Regular`）。
- 验证：`latexmk -g -xelatex -interaction=nonstopmode -halt-on-error -outdir=build main.tex` 可通过。
- 建议处理：
  1. 在 `thesis_overleaf/README` 或项目文档里补“编译环境依赖清单（TeX + 字体）”。
  2. 在答辩前做一次“全新环境冷启动编译”验证，降低换机风险。

### 2.2 系统默认权重策略仍需文档与配置双落地

- 现状：论文口径与系统口径已有区分，但 `global-best` 与 `stable-best` 的默认策略仍需进一步固化。
- 影响：答辩问到“为什么系统默认不是论文最优权重”时，需要更直接的配置证据与说明。
- 建议处理：
  1. 在 `configs/models.yaml` 明确每模型默认用途（论文复现实验 / 系统演示）。
  2. 在 `docs/system/artifact_integration_spec.md` 增加一张“权重选择规则表”。

### 2.3 任务演示样例库尚未结构化归档

- 现状：已有可用图片与历史任务结果，但“易/中/难”固定样例池没有标准化目录和说明。
- 影响：现场演示的稳定性与复现性依赖临场选择。
- 建议处理：
  1. 建立 `assets/demo_cases/{easy,medium,hard}`。
  2. 每类给出 3~5 张固定图及预期讲解点。
  3. 在 `docs/midterm/` 或 `docs/system/` 增补一页“演示脚本+兜底截图映射”。

## 3. 中优先级缺口（建议本周收口）

### 3.1 参考文献库仍有模板遗留条目

- 现状：`hfut.bib` 可用，但存在与正文无关的模板项。
- 建议处理：
  1. 清理明显无关项或分离到 `hfut_template_unused.bib`。
  2. 对正文实际 `\cite{}` 做一次“key 存在性 + 去重”扫描。

### 3.2 FastAPI 生命周期写法有弃用告警

- 现象：测试通过，但有 `on_event` deprecation warning。
- 建议处理：
  1. 迁移到 lifespan 事件写法。
  2. 保持行为不变，仅做兼容性升级，避免后续版本风险。

### 3.3 插件化部署还缺“打包级”规范

- 现状：已支持 `pkg.module:factory`，比绝对路径规范很多。
- 仍缺：部署时如何确保插件包随环境安装（`pip install -e .` 或 wheel）。
- 建议处理：
  1. 在部署文档给出“开发态/部署态”两套插件加载建议。
  2. 在 `README` 增加最小可执行示例（从空环境到跑通）。
