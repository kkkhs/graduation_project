## 系统需求分析（开题/论文第 5 章可直接使用）

### 1. 系统目标
构建“遥感微小船舶检测系统”原型：集成 ≥3 种检测模型，实现遥感图像推理、结果可视化与结构化导出，为答辩演示与工程验证提供可运行载体。

### 2. 术语与定义
- **推理（Inference）**：对输入图像运行模型，输出 bbox/score。
- **可视化**：在原图叠加 bbox 与置信度等信息。
- **结构化结果**：JSON/CSV 等可用于后处理或统计分析的输出。

### 3. 功能需求（Functional Requirements）
#### 3.1 模型管理
- FR-1：支持选择模型（至少 3 个：DRENet、mmdetection 模型、YOLO 模型）。
- FR-2：支持加载本地权重与配置（如 `.pth`/`.pt`/`onnx`）。

#### 3.2 推理与评测
- FR-3：支持单张图像推理（输入文件路径）。
- FR-4：支持批量推理（输入文件夹，输出文件夹）。
- FR-5（可选）：支持对测试集运行评测并输出指标汇总（mAP/AP50 等）。

#### 3.3 输出与导出
- FR-6：输出可视化结果图（bbox+score）。
- FR-7：导出结构化结果（JSON/CSV），包含：image_id、bbox、score、category。
- FR-8：记录运行参数（model、conf、iou、输入尺寸等）便于复现。

#### 3.4 交互方式
- FR-9：提供 CLI 命令行方式（必须，便于复现与批处理）。
- FR-10（可选）：提供 Web Demo（建议 Gradio/Streamlit，用于答辩演示）。

### 4. 非功能需求（Non-functional Requirements）
- NFR-1：可复现安装（环境说明、依赖版本、运行步骤）。
- NFR-2：优先支持 GPU 推理；CPU 作为备选（可声明性能差异）。
- NFR-3：稳定性：对异常输入给出清晰错误提示，不崩溃。
- NFR-4：可维护性：模型适配层统一接口，便于新增/替换模型。
- NFR-5：性能目标（可写期望）：在推荐 GPU 上单张推理耗时 \<\<填写\>\> 秒以内。

- **最低**：CPU 可运行（仅用于功能验证/小规模推理）
