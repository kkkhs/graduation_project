\chapter{总结与展望}
\label{ch:conclusion}

\section{工作总结}

本文围绕“遥感场景下的微小船舶检测系统设计与实现”这一课题，完成了文献调研、方法对比、系统实现和论文撰写的阶段性工作。首先，本文梳理了通用目标检测、小目标检测和遥感船舶检测的研究进展，明确了当前研究在统一评测、可复现性和系统落地方面的不足。其次，本文以 LEVIR-Ship 数据集为统一基准，选择 DRENet、FCOS 和 YOLO26 三类代表模型开展了对比实验，并在统一实验口径下分析了各模型在精度、召回和部署性方面的差异。

从当前实验结果看，DRENet 在召回方面表现突出，说明其对复杂背景中的微小目标具有较强检出能力；YOLO26 在 AP50、AP50-95、Precision 和 F1 等指标上表现均衡，体现出较强的工程适用性；FCOS 则提供了 anchor-free 路线下的重要基准参照。以上结果说明，不同检测范式在遥感微小船舶任务中各有优势，模型选择应结合具体场景需求，而不应只依据单一指标作出结论。

在系统实现方面，本文基于 React、FastAPI 和 SQLite 搭建了轻量级检测系统，打通了“任务提交—模型推理—结果可视化—历史回看”的完整链路。系统已具备前后端、数据库和算法集成的基本完整性，能够支撑毕业设计答辩中的功能演示和结果展示。

\section{不足分析}

尽管本文已完成课题主线工作，但仍存在一些不足：

\begin{enumerate}
  \item 当前三模型的效率指标，如 FPS、参数量和 FLOPs，仍需进一步统一测量并回填；
  \item 消融实验与难例图集尚未完全补齐，导致对部分性能差异的解释还不够充分；
  \item FCOS 的精细指标和稳定性分析仍有补充空间；
  \item 系统默认权重策略和插件化部署规范仍可进一步完善，以提升跨机器部署的规范性。
\end{enumerate}

\section{未来展望}

后续工作可从以下几个方向继续展开：

\begin{enumerate}
  \item \textbf{补齐实验闭环}：进一步完善效率指标、消融实验和定性分析，形成更完整的论文实证部分；
  \item \textbf{优化模型性能}：围绕输入分辨率、增强策略、阈值设置和融合策略开展进一步调优；
  \item \textbf{增强系统能力}：在现有单机系统基础上，探索更加规范的模型插件化接入与部署方式；
  \item \textbf{扩展应用场景}：将现有方法迁移到更大规模、更复杂场景的数据集上，验证模型泛化能力。
\end{enumerate}

\section{全文结论}

总体来看，本文完成了一个具有明确问题背景、统一实验协议、真实系统实现和可追溯结果支撑的本科毕业设计工作。研究表明，面向遥感微小船舶检测任务，仅关注算法精度是不够的，必须同时考虑召回能力、工程部署难度和系统可演示性。本文的实验结果与系统实现为后续进一步优化提供了基础，也为类似场景下的本科毕业设计提供了可参考的实现路径。
