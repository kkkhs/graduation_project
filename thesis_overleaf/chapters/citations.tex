\chapter{实验结果与分析}
\label{ch:experiment}

\section{实验环境与设置}

本文实验围绕 LEVIR-Ship 数据集展开，采用统一的数据划分和评测口径，对 DRENet、FCOS 与 YOLO26 三类代表模型进行对比分析。核心评价指标包括 AP50、AP50-95、Precision、Recall 与 F1；为辅助分析工程部署能力，还统计了 FPS、参数量与 FLOPs 等效率指标。实验过程中，训练配置、权重文件、运行日志和结果文档均进行了统一留痕，以保证论文表格和系统演示能够回溯到具体实验产物。

需要特别说明的是，DRENet 与 YOLO26 采用统一的 512 输入主线，而 FCOS 当前可用的正式结果来自 MMDetection 默认训练口径，其输入策略为 1333\(\times\)800。基于这一客观差异，本文对三模型的使用口径作出如下区分：DRENet 与 YOLO26 作为论文主结论模型进行重点比较；FCOS 作为 anchor-free 参考基线参与对比，用于补充不同检测范式的讨论，而不用于强化“同输入尺度严格公平”的结论。

从工程角度看，本文不仅保留了训练与测试结果，也完成了系统侧的真实推理联调。这意味着本章中的定量分析并非孤立存在，而是能够与第五章中的系统实现相互印证。

FCOS 的 AP50 与 AP50-95 均低于 DRENet 和 YOLO26，这说明在当前实验设置下，anchor-free 通用基线在遥感微小船舶场景中的整体表现尚不占优。但 FCOS 仍然具有参考意义：它为本文提供了与专项方法、one-stage 方法不同的检测范式参照，也为后续分析“是否必须采用专项设计”提供了比较基础。

\section{模型逐项分析}

\subsection{DRENet 结果分析}

DRENet 是面向遥感微小船舶任务的专项方法，其最突出特点体现在较高的 Recall 指标上。当前实验中，DRENet 的 Recall 达到 0.8511，明显高于 YOLO26，这说明 DRENet 在复杂背景与弱纹理目标场景下更倾向于给出候选框，从而减少漏检。对于海上监管、目标排查等更重视“尽可能发现目标”的任务，这种特性具有实际价值。

但与此同时，DRENet 的 Precision 为 0.4927，低于 YOLO26。这表明其在提升召回的同时也引入了较多误检，特别是在近岸背景、港口设施和纹理复杂区域中更容易受到干扰。因此，DRENet 更适合作为强调召回优先的主方法，若要进一步用于实际部署，还需要结合阈值调优、后处理或多模型融合来抑制误检。

\subsection{FCOS 结果分析}

FCOS 作为本文选取的 anchor-free 代表模型，其 AP50 为 0.7700，整体结果弱于 DRENet 与 YOLO26。从最终表现看，通用 anchor-free 路线在当前中分辨率微小船舶场景中的优势并不明显，至少在本文现有实验条件下尚未体现出超越专项方法或成熟 one-stage 模型的能力。

不过，FCOS 的意义并不在于追求最佳结果，而在于提供一个结构清晰、范式明确的参考基线。通过引入 FCOS，本文能够更完整地回答“专项设计是否必要”“通用 anchor-free 方法在该任务中能达到怎样的水平”等问题。结合训练记录看，FCOS 也为后续更规范的框架复现、输入策略分析和稳定性讨论保留了空间。

\subsection{YOLO26 结果分析}

YOLO26 在当前实验中取得了最优的 AP50-95、Precision 和 F1，同时 AP50 与 DRENet 基本持平，说明其在检测精度、定位质量和误检控制之间表现出较好的综合平衡。尤其是 Precision 达到 0.8430，说明在当前阈值设置下，YOLO26 对复杂背景下的伪目标抑制能力较强。

从工程角度看，YOLO26 还具有训练链路成熟、部署接口清晰和系统接入成本较低等优势。因此，若从“本科毕设需要稳定可演示系统”的角度考虑，YOLO26 是很有竞争力的默认模型候选。与 DRENet 结合时，YOLO26 可以在整体精度和误检控制方面提供互补。

\section{定性样例与误差分析}

\section{消融实验}

为进一步增强主结论的可解释性，本文补充两组离线消融分析。第一组为三模型统一的置信度阈值消融，即在固定 IoU 阈值下调节置信度阈值，比较 DRENet、FCOS 和 YOLO26 在 Precision 与 Recall 平衡上的变化趋势；第二组为输入尺寸敏感性分析，即在不重新训练模型的前提下，仅改变推理阶段输入尺寸，观察模型对小目标尺度放大的响应差异。需要说明的是，输入尺寸组属于推理阶段敏感性验证，不等同于重新训练后的完整尺寸消融，其主要作用是解释不同模型在现有正式权重基础上对输入尺度扰动的行为差异，而不是反向覆盖前文主实验结论。

\subsection{阈值消融分析}

阈值消融统一固定 IoU 阈值为 0.50，仅扫描置信度阈值 0.15、0.25 与 0.35。该组结果主要用于回答两个问题：其一，不同模型在低阈值和高阈值下的 Precision 与 Recall 变化是否具有一致规律；其二，系统默认阈值为何选择 0.25，而不是更低或更高的设置。需要特别说明的是，FCOS 在本节消融中统一使用系统联调与离线复算所采用的稳定候选口径对应 checkpoint，而不是主对比表中的历史最佳单点结果。三模型离线消融结果如表~\ref{tab:ch4_threshold_ablation} 所示。

从表~\ref{tab:ch4_threshold_ablation} 可以看出，三模型均表现出较一致的趋势：随着阈值升高，Precision 普遍上升，而 Recall 普遍下降。这说明在当前遥感微小船舶检测任务中，后处理阈值仍然是影响检出行为的重要因素。对于 DRENet 而言，较低阈值下能够进一步放大其“召回优先”的特点，但也会引入更多误检；当阈值提高到 0.35 时，Precision 提升到 0.7778，而 Recall 下降到 0.7355，说明 DRENet 的输出确实更依赖阈值来抑制背景干扰。综合考虑后，0.25 仍是更适合作为系统默认值的折中选择。

FCOS 与 YOLO26 的阈值变化同样体现出不同的范式差异。FCOS 在本组离线评测中，当阈值由 0.25 提高到 0.35 时，F1 从 0.7944 小幅提升到 0.7986，说明其在当前稳定权重下对阈值较为敏感，高阈值能够进一步压缩冗余候选框。YOLO26 则在 0.25 时取得更均衡的 AP50、Precision 与 Recall 组合；继续提高到 0.35 虽然能够将 Precision 提升到 0.8248，但 AP50 降至 0.5914，Recall 也明显下降。因此，从系统默认配置和答辩解释的角度出发，YOLO26 采用 0.25 更便于兼顾精度与召回，而 DRENet 更能体现“尽量检出目标”的路线特征。

\subsection{输入尺寸敏感性分析}

输入尺寸敏感性分析用于考察不同模型在推理阶段对目标尺度放大的依赖程度。本组实验不重新训练模型，只对推理输入尺寸进行覆写，因此它反映的是“现有权重面对尺度扰动时的行为差异”，而非严格意义上的训练尺度消融。对 FCOS 而言，其主对比参考结果来自历史正式 run 的最佳点，而本节尺寸分析则统一基于稳定候选口径对应 checkpoint 进行离线复算，因此这些结果仍仅用于行为解释，不反向改写前文主对比结论。YOLO26 与 FCOS 的结果如表~\ref{tab:ch4_size_ablation} 所示。

由表~\ref{tab:ch4_size_ablation} 可以看出，FCOS 对输入尺寸高度敏感。当输入尺寸由 512 提升到 640 时，其 AP50 由 0.4490 提升到 0.6452，F1 也由 0.6139 提升到 0.7601；进一步增大到 800 后，AP50 与 F1 分别达到 0.7389 和 0.7944。该趋势说明 FCOS 对微小目标可见性的依赖较强，较大的推理输入能够显著改善其候选生成与定位质量。也正因如此，FCOS 更适合作为“输入尺度变化敏感”的参考基线，而不是简单与 512 口径下的模型做表面比较。

YOLO26 的尺寸变化趋势则体现出另一种更偏工程部署的特性。与 FCOS 持续受益于更大输入不同，YOLO26 在 640 时取得了最优的 AP50 和 F1，表现出更好的综合平衡；继续放大到 800 虽然 Recall 从 0.7953 进一步提高到 0.7989，但 Precision 从 0.7543 下降到 0.6827，F1 也下降到 0.7362。这表明 YOLO26 在更大输入下虽然能够看到更多候选目标，但也更容易引入额外误检。因此，从默认部署参数角度看，YOLO26 并不是输入越大越好，而是需要在检出率和误检控制之间寻求折中，这也与前文将其作为系统默认模型候选的结论保持一致。

对于 DRENet，本组未将其纳入统一尺寸主表。原因并非实验未完成，而是当前本地插件实现路径在非 512 输入下会触发结构 shape mismatch，导致 640 和 800 设置无法稳定输出结果。因此，本文将该现象明确写为当前实现边界，而不是将其误写为“结果缺失”。这一处理方式能够保证论文本身的诚实性与可复现性，也与系统实现部分对本地插件约束的描述保持一致。

此外，本文还在 20 张图子样本上进行了 smoke\_limit 补充验证。该组结果仅用于说明全量结果中的尺寸趋势在小样本上仍具有一致方向，不进入正文主表，以避免与全量测试集结果混用。

\section{系统口径与论文口径说明}

为了保证论文论证与系统实现之间的一致性，本文对模型使用口径进行了专门区分。在论文主结论中，重点使用 DRENet 与 YOLO26 的正式结果，并将 FCOS 保留为参考基线；在系统实现中，除了单模型推理外，还提供融合模式和默认权重配置，用于增强系统演示时的结果完整性。对于 FCOS，本文统一采用“论文主对比口径”和“系统与消融口径”两类表述：前者对应历史正式 run 中用于论文主表展示的最佳离线结果，后者对应系统联调、定性导出与本章离线消融所使用的稳定候选 checkpoint。

这种区分并不意味着论文和系统相互割裂，而是强调二者服务于不同目标：论文更关注实验结论的解释力和主表口径一致性，系统更关注任务执行的稳定性与展示效果，而消融实验更关注在统一离线复算条件下观察参数扰动后的行为变化。明确这一点，有助于在答辩中更清楚地解释为何 FCOS 在主对比、系统展示和消融分析中会出现不同用途但不相互冲突的结果来源。

\section{阶段性不足与后续补充方向}

尽管本文已经形成较完整的主对比结果、定性样例和两组离线消融，但实验部分仍存在若干阶段性不足。首先，当前输入尺寸敏感性分析只覆盖了 YOLO26 与 FCOS；DRENet 在本地插件路径下使用非 512 输入会触发结构 shape mismatch，因此该部分以“实现边界说明”形式保留，而未强行纳入统一主表。其次，FCOS 的效率结果与正式参考结果由于输入策略不同，只能作为参考值，而不能与 DRENet 和 YOLO26 做完全同口径比较。再次，系统中的融合模式已经能够稳定运行，但目前尚未形成统一离线 AP 指标表，因此本文暂不把融合结果作为主结论展开论证，而是主要将其用于系统展示和结果互补分析。

对于本科毕业设计而言，以上不足并不会破坏整篇论文的完整性，因为主结论已经由 DRENet 与 YOLO26 的正式结果支撑，FCOS 的角色也已明确界定，阈值消融与尺寸敏感性分析也已经能够支撑“行为差异解释”这一实验目的。后续若有进一步时间和算力条件，可围绕融合离线评估、更严格统一口径效率测试以及 FCOS 同尺度训练复现继续完善。

\section{本章小结}

本章在统一评测协议下，对 DRENet、FCOS 和 YOLO26 三种模型进行了定量与定性分析，并进一步通过阈值消融与输入尺寸敏感性分析补充了主结论的可解释性。结果表明，DRENet 在召回方面表现突出，更适合强调漏检控制的场景；YOLO26 在 AP50、AP50-95、Precision 和 F1 等指标上表现均衡，更适合作为轻量部署与系统默认模型候选；FCOS 则提供了 anchor-free 路线的参考基线，并通过稳定候选口径下的离线消融展示了其对阈值与输入尺寸的敏感性。结合效率分析、误差样例和系统口径说明，可以认为本文已经形成较完整的实验论证框架，为下一章的系统设计与实现提供了直接依据。
